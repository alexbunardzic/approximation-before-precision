# Testing Strategy

Test Layers:
1. Architecture tests
2. Domain microtests
3. Mutation testing

Goal:
Uncover domain rules as tests keep growing. Prevent regression + enforce design integrity

Ground rule: 
Always make tests as simple as possible. 
No conditional logic in any tests. 
If you ever get an idea to modify any test, stop and let me know why you feel you need ot modify a passing test, and wait for my explicit approval.
